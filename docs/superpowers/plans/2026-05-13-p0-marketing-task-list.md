# P0 Marketing Task List

> **For agentic workers:** Use this as an execution checklist for the product's first marketing loop. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Validate that early users want a desktop AI companion because it feels present, calming, and different from a normal chat window.

**Marketing Thesis:** We are not selling "Live2D + AI." We are selling "桌面上多了一个会陪你的存在感".

**Primary Positioning:** 一个住在桌面上的 AI 小伙伴。

---

## Core Rule

Every public-facing asset should make at least one of these points obvious within seconds:

1. 她住在桌面上
2. 她会有反应
3. 她不是普通聊天机器人
4. 她会记住你一点点
5. 她需要时在，专注时安静

If an asset only demonstrates "问一句，答一句", it is too generic.

---

## Task 1: Freeze P0 Messaging

**Goal:** Make sure every future asset uses the same product story.

- [ ] Confirm the primary one-liner:
  - `一个住在桌面上的 AI 小伙伴。`
- [ ] Confirm the expanded description:
  - `不是聊天窗口，而是住在你桌面上的 AI 小伙伴。`
  - `她会陪你聊天、会有表情动作、会记住你一点点，也会在你专注时安静陪着你。`
- [ ] Define the 3 strongest value tags for P0:
  - `桌面存在感`
  - `轻陪伴`
  - `低打扰`
- [ ] Define words to avoid in first-wave messaging:
  - `平台`
  - `全球领先`
  - `革命性`
  - `情感治疗`
  - `高度智能人格`

**Deliverable:**
- One short internal message guide that copywriting, video scripts, landing page, and alpha posts all reuse.

**Success criteria:**
- A stranger can understand the product without hearing any technical stack terms.

---

## Task 2: Define the First 30 Seed Users

**Goal:** Narrow the first audience so the alpha learns from the right people.

- [ ] Prioritize these seed groups in order:
  1. 二次元 / Live2D / 桌宠用户
  2. AI 工具重度用户
  3. 长时间桌面工作或创作的人
- [ ] Write a one-line reason for each group:
  - why they would install
  - why they might keep it open
  - what they can teach us
- [ ] Define what not to target in P0:
  - broad productivity users with zero interest in character presence
  - users expecting therapy or deep emotional reliance
  - users demanding a full automation agent product
- [ ] Write the first-wave tester filter questions:
  - 你是 Windows 还是 macOS 用户？
  - 你是否用过桌宠 / Live2D / AI 聊天？
  - 你更想要陪伴感还是效率辅助？
  - 你是否愿意反馈 bug 和体验问题？
  - 你是否愿意加入反馈群？

**Deliverable:**
- A short alpha screener form outline.

**Success criteria:**
- The first 20-50 users are likely to care about presence, not just utility.

---

## Task 3: Produce the First Demo Asset Pack

**Goal:** Create the minimum visual proof needed for people to "get it".

Source script file: `docs/marketing/p0-demo-scripts.md`

- [ ] Record one 15-second presence-first demo:
  - desktop idle state
  - click character
  - ask a light emotional or practical question
  - show thinking state
  - show reply state
  - return to calm idle
- [ ] Record one 15-second efficiency-first demo:
  - user working
  - gets stuck
  - clicks companion
  - asks for help
  - receives reply
  - continues working while character stays quiet
- [ ] Record one short character-reaction demo:
  - idle
  - listening
  - thinking
  - talking
  - hide / quiet presence
- [ ] Export at least:
  - 1 short MP4 for B站 / 小红书 / 抖音
  - 1 GIF for quick sharing
  - 3 screenshots for landing page or signup page

**Deliverable:**
- A small demo asset pack with 3 short clips and 3 screenshots.

**Success criteria:**
- Someone seeing the video without context can tell this is not just a browser chat product.

---

## Task 4: Create the Alpha Signup Page

**Goal:** Turn curiosity into a small waitlist of motivated testers.

- [ ] Create a simple signup page or form using one of:
  - Tally
  - Notion
  - Feishu form
  - a simple landing page in the repo later if needed
- [ ] Include these sections only:
  - product one-liner
  - 15-second demo video or GIF
  - who this is for
  - current stage: early alpha
  - what kind of feedback is needed
  - signup button
  - group/join instructions
- [ ] Add one expectation-setting sentence:
  - `这是一个很早期的版本，可能会有 bug，但我会根据反馈快速迭代。`
- [ ] Add one privacy/trust sentence:
  - `她只会记住少量你明确可见和可删除的信息。`

**Deliverable:**
- One alpha signup page or form link.

**Success criteria:**
- Interested users can understand the product and register in under 60 seconds.

---

## Task 5: Prepare the First-Wave Distribution Posts

**Goal:** Launch 3-5 posts with different angles to see what resonates.

- [ ] Write one `陪伴感` version post.
- [ ] Write one `效率助手` version post.
- [ ] Write one `角色反应感` version post.
- [ ] Adapt each post for these channels:
  - 小红书
  - B站
  - 即刻
  - V2EX
  - QQ 群 / Discord
- [ ] Keep channel-specific differences clear:
  - 小红书: 治愈感、桌面美化、陪伴
  - B站: 演示视频、开发日志、完整体验
  - 即刻: 独立开发 + 内测招募
  - V2EX: 产品逻辑 + 桌面产品讨论
  - QQ/Discord: 兴趣社群内测招募

**Deliverable:**
- A post pack with 3 core post variants and channel-adjusted versions.

**Success criteria:**
- You can publish to 3-5 channels in one batch without improvising copy each time.

---

## Task 6: Build the Alpha Feedback Loop

**Goal:** Convert installs into structured product learning.

- [ ] Create one feedback group:
  - QQ 群 / Discord / 微信群 / Telegram, choose one primary home
- [ ] Prepare a welcome message that says:
  - what this product is
  - what stage it is in
  - what kind of feedback is useful
  - how often updates will happen
- [ ] Prepare a repeatable daily/weekly prompt set:
  - 今天有没有打开它？
  - 哪个瞬间让你觉得她“活了”？
  - 哪个地方最烦？
  - 你更把她当工具还是伙伴？
  - 如果只能保留一个功能，你会保留什么？
- [ ] Prepare a simple bug / suggestion format:
  - 发生了什么
  - 你当时在做什么
  - 是否能稳定复现
  - 你原本希望发生什么

**Deliverable:**
- One feedback group and one reusable question loop.

**Success criteria:**
- Feedback becomes structured enough to separate novelty from real retention drivers.

---

## Task 7: Define the P0 Metrics Dashboard

**Goal:** Track whether the product is becoming something users voluntarily keep around.

- [ ] Track installation success.
- [ ] Track first successful launch.
- [ ] Track chat success rate.
- [ ] Track history restore success.
- [ ] Track D1 and D3 reuse in the alpha group.
- [ ] Track average daily conversation count.
- [ ] Track whether users leave the app open for long sessions.
- [ ] Track user quotes in 3 categories:
  - presence
  - comfort / companionship
  - annoyance / friction
- [ ] Define the weekly review questions:
  - Did users come back?
  - Did they describe her as present?
  - What made the product feel real?
  - What made it feel gimmicky?

**Deliverable:**
- One lightweight weekly P0 review template.

**Success criteria:**
- The team can tell whether presence is real without hiding behind vanity metrics.

---

## Task 8: Decide the First Public Narrative Loop

**Goal:** Create an ongoing content rhythm instead of one-off posting.

- [ ] Commit to one recurring content stream:
  - dev log
  - short weekly build update
  - before/after interaction improvements
- [ ] Define 5 reusable post formats:
  - `我做了一个住在桌面上的 AI 小伙伴`
  - `普通聊天窗口 vs 桌面 AI 伙伴`
  - `这个动作做自然以后，感觉完全变了`
  - `第一批内测用户最喜欢的 3 个点`
  - `我们砍掉了一个很烦的主动行为`
- [ ] Keep one rule for every update:
  - show visible experience change, not only architecture or code
- [ ] Write one sentence that guides all future public updates:
  - `我们不是在堆 AI 功能，而是在打磨“她在桌面上陪着你”的感觉。`

**Deliverable:**
- One repeatable content cadence for the alpha period.

**Success criteria:**
- Marketing becomes a continuous learning loop, not a one-time launch event.

---

## Priority Order

Execute in this order:

1. Task 1: Freeze messaging
2. Task 2: Define seed users
3. Task 3: Produce demo asset pack
4. Task 4: Create alpha signup page
5. Task 5: Prepare distribution posts
6. Task 6: Build feedback loop
7. Task 7: Define metrics dashboard
8. Task 8: Decide public narrative loop

---

## P0 Exit Criteria

This marketing task list is complete when:

- the product has one clear sentence people repeat back,
- 20-50 seed users can understand it and sign up,
- at least one demo asset reliably communicates presence,
- feedback arrives in a structured way,
- and the team can judge whether users want companionship, utility, or both.
